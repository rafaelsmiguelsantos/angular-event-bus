Services/Comunicação por componentes


  • Um "Event Bus" pode fornecer uma maneira fracamente acoplada de comunicação entre objetos.
  • Os serviços podem expor um Subject/BehaviorSubject/ReplaySubject que outros serviços/componentes podem assinar para alterações.
  • Soluções de gerenciamento de estado também podem ser usadas.

  



Notas:

  • Este exemplo mostra como serviços e assuntos podem ser usados para fornecer um mecanismo de comunicação para componentes. Clique em um dos botões abaixo e observe como os clientes contam o número no canto superior direito mudanças (o barramento de eventos é usado). Observe que quando um cliente é selecionado no componente de apresentação "Clientes" os detalhes são exibidos e os detalhes do cliente são exibidos abaixo da barra de navegação (BehaviorSubject é usado)..
  • O botão "Adicionar cliente (somente push)" chamará um serviço para adicionar um cliente, mas os dados de clientes retornados não serão exibidos corretamente e não acionarão ngOnChanges no componente de apresentação, pois a matriz de clientes em si não muda.
